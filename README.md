# TaskWeave 项目架构计划书

## 1. 项目定位

TaskWeave 是一个面向桌面端的 Python 定时任务执行管理工具，主要用于统一管理本地 Python 脚本、目录任务和定时调度规则。项目采用 Go 作为主要开发语言，使用 Wails 构建跨平台桌面 GUI，调度能力基于 `robfig/cron`，Python 任务运行统一通过 `uv` 进行环境和依赖管理。

项目支持 Linux、macOS、Windows 桌面平台，不支持移动端。最终交付形态为可安装、可打包的桌面应用。

## 2. 建设目标

- 提供桌面 GUI，用于创建、编辑、启停和查看 Python 定时任务。
- 支持单个 Python 文件、多个 Python 文件和目录级任务配置。
- 使用 `uv` 管理 Python 任务的依赖、虚拟环境和执行入口。
- 使用 `robfig/cron` 提供稳定的本地定时调度能力。
- 支持任务执行日志、运行状态、最近执行结果和错误信息查看。
- 支持跨平台打包，覆盖 Linux、macOS、Windows。

## 3. 技术选型

| 层级 | 技术 | 用途 |
| --- | --- | --- |
| 桌面壳 | Wails | 构建跨平台桌面应用，连接 Go 后端与前端 UI |
| 后端核心 | Go | 任务配置、调度、进程管理、日志管理、跨平台能力 |
| 调度器 | `robfig/cron` | cron 表达式解析和定时任务触发 |
| Python 运行 | `uv` | Python 包管理、虚拟环境管理、任务启动 |
| 前端 | React + TypeScript | 任务管理界面、日志界面、设置界面 |
| 前端工具链 | Bun + Vite | 前端依赖安装、开发服务和生产构建 |
| 配置存储 | JSON 文件 | 保存任务配置和应用设置 |
| 日志存储 | log 文件 | 保存任务执行输出和执行记录 |

### 3.1 前端与桌面技术选择

项目桌面端采用 `Wails + React + Vite + Bun` 的组合：

- `Wails` 负责桌面应用容器、跨平台打包，以及 Go 后端与前端 UI 之间的绑定通信。
- `React` 负责构建任务列表、任务编辑、日志查看和应用设置等交互界面。
- `Vite` 作为前端开发和构建工具，提供快速热更新和生产构建能力。
- `Bun` 作为前端包管理和脚本运行工具，用于依赖安装、开发服务启动和构建命令执行。

## 5. 核心模块规划

### 5.1 GUI 模块

负责提供用户可视化操作入口，主要页面包括：

- 任务列表：展示任务名称、启用状态、cron 表达式、最近执行时间和最近执行结果。
- 任务编辑：配置任务名称、任务类型、文件路径或目录路径、启动命令、cron 表达式和环境变量。
- 执行日志：查看任务标准输出、标准错误、退出码、开始时间、结束时间和执行耗时。
- 应用设置：配置 `uv` 路径、默认工作目录、日志保留策略和开机启动等选项。

### 5.2 任务配置模块

负责维护任务定义，建议任务模型包含：

- `id`：任务唯一标识。
- `name`：任务名称。
- `type`：任务类型，支持 `file`、`multi_file`、`directory`。
- `path`：Python 文件或目录路径。
- `entry`：目录任务的入口文件或启动命令。
- `cron`：定时调度表达式。
- `enabled`：是否启用。
- `env`：任务级环境变量。
- `workingDir`：任务工作目录。
- `createdAt` / `updatedAt`：创建和更新时间。

### 5.3 调度模块

调度模块基于 `robfig/cron` 实现，主要职责包括：

- 加载已启用任务并注册到 cron 调度器。
- 在任务新增、修改、删除、启停时动态更新调度规则。
- 支持应用启动后自动恢复任务调度。
- 避免同一任务在上一次执行未结束时重复启动，可提供跳过、并发、排队三种策略。

### 5.4 执行模块

执行模块负责调用 `uv` 启动 Python 任务，主要职责包括：

- 根据任务类型生成执行命令。
- 设置工作目录、环境变量和进程上下文。
- 捕获标准输出、标准错误和退出码。
- 支持手动运行和定时运行。
- 支持任务取消或终止。

建议执行方式：

```text
uv run python script.py
uv run python main.py
uv run <custom-command>
```

目录任务可根据配置选择入口文件或自定义命令，避免强绑定单一执行方式。

### 5.5 日志与执行记录模块

负责记录任务执行过程和结果，主要数据包括：

- 任务 ID。
- 触发方式：手动或定时。
- 开始时间、结束时间、执行耗时。
- 执行状态：运行中、成功、失败、已取消、已跳过。
- 标准输出、标准错误。
- 退出码和错误摘要。

日志使用独立 log 文件保存，建议按任务和日期进行分片，避免单文件无限增长。执行记录可以写入日志索引文件或任务级记录文件，便于 GUI 快速展示最近执行状态。

### 5.6 存储模块

项目采用本地文件存储，不引入 SQLite。配置数据使用 JSON 文件保存，任务执行输出和执行记录使用 log 文件保存。

建议分层：

- 配置存储：使用 JSON 文件保存任务定义和应用设置。
- 执行记录存储：使用任务级或日期级 log 文件保存执行历史摘要。
- 日志文件存储：使用独立 log 文件保存标准输出、标准错误和错误摘要。
- 文件索引：可使用轻量 JSON 索引记录最近运行状态、日志路径和最后执行时间。
